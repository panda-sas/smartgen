# SmartGen Examples

This directory contains example projects generated by SmartGen, demonstrating how to use the tool to create DDD-compliant domain code.

## Available Examples

### E-Commerce Domain

A complete e-commerce domain with:
- User management (registration, profiles, roles)
- Product catalog (products, categories, inventory)
- Shopping cart functionality
- Order processing and management

**Location**: [`ecommerce/`](ecommerce/)

**See**: [`ecommerce/readme.md`](ecommerce/readme.md) for detailed documentation

## How to Use Examples

1. **Review the SRS**: Each example includes a `srs.md` file showing the requirements
2. **Examine Generated Code**: See how SmartGen translates requirements into DDD code
3. **Learn Patterns**: Study the generated code to understand DDD patterns
4. **Adapt for Your Project**: Use examples as templates for your own domains

## Example Structure

Each example follows this structure:

```
example-name/
├── srs.md              # Software Requirements Specification
├── readme.md           # Example documentation
└── src/
    └── domain/         # Generated domain layer
        ├── aggregates/
        ├── entities/
        └── value_objects/
```

## Generating Your Own Examples

To generate a similar example:

1. **Create SRS**: Write your requirements in `srs.md`
2. **Initialize Project**: Run `smartgen init`
3. **Generate Domain**: Run `smartgen generate domain`
4. **Review Code**: Check the generated code in `src/domain/`

## Contributing Examples

We welcome example contributions! If you have a domain example that demonstrates SmartGen well:

1. Create a new directory under `examples/`
2. Include a complete `srs.md`
3. Include generated domain code
4. Add a `readme.md` explaining the example
5. Submit a pull request
